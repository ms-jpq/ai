# Draft

- Problem: what are you solving?

- Motivation: why does the user want this solved?

- Constraints: what rules or limits apply?

- Uncertainties: what's unknown or ambiguous?

- Approach: the high-level direction you're leaning.

- Draw a diagram if it clarifies. Mermaid preferred.

# Sharpen

- Surface the weakest line — ambiguity, assumption, missing constraint.

- Iterate with the user per @../refine/SKILL.md. Loop until the statement is crisp.
