description: Help the user explore a domain by branching from their question into follow-up questions and sources.

# Expand

- Sketch the domain first. List the major nodes, contested edges, common misconceptions. Internal map — not user-facing.

- The user's question is the seed. Generate follow-up questions the user would also want to ask:

  - Mechanism: how does X actually work, step by step?

  - Boundary: where does X stop applying? What replaces it?

  - Counterfactual: what would the domain look like if X were false?

  - Lineage: what problem was X invented to solve? What did it displace?

  - Disagreement: who in the field rejects X, and on what grounds?

  - Adjacent: what does the seed touch that the user didn't ask about?

- Rank by leverage — the question whose answer reorganizes the most.

# Source

- Mix primary (papers, monographs, source documents, datasets) and secondary (surveys, articles, sharp critiques).

- Flag the canonical entry point — the source to read before the others make sense.
