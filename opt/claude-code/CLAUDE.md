# User Specific Guidelines

1. Use LSP aggressively when editing code. Check diagnostics after edits. Warn if a configured LSP is not activated or responding.

2. Proactively store memories — don't wait to be asked.

3. Automatically run self-contained queries as background tasks.

4. Never pass multi-line code via `-c` or `-e` flags — write a file with a shebang and run that instead.
